help:

  1. How to use a IndexName of token to represent nullptr in #issues/18149? https://github.com/pingcap/parser/blob/master/parser.y#L4940 https://github.com/mysql/mysql-server/blob/8.0/sql/sql_yacc.yy#L7456

  2. IndexName，属于类型，是字符串类型，无法表示nil，我对比mysql的，mysql是自定义字符串类自己区分了。https://github.com/mysql/mysql-server/blob/8.0/sql/sql_yacc.yy#L7456。 ，问：用"nullptr"字符串表示nil，发现影响其他测试用例。IndexName 属于ident>类型，是字符串类型不能用nill表示？

老王：

  1. How about define a struct NullIndexName like sql.NullString : type NullString struct { String string Valid bool // Valid is true if String is not NULL }

  2. We can use a struct or a nil pointer to represent a empty index to distinguish from a anonymous index. To record this message to AST, a new type like ConstraintEmptyIndex can be added to ConstraintType . Then TiDB can throw the error (1280, “Incorrect index name ‘'") at the time of handling constraints.

所以, 这个地方有两种出路, 一个是改变indexName的返回值, 你返回些别的信息, 一个结构体什么的, 但是得改所有相关处 另一个是就用类似的字符串方式, 使用某个保留关键字之类的做当作nil, 也就是不允许作为IndexName的关键字… 这样改动比较少一点, 但是关键字得好好选
